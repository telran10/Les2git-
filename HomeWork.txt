 1.Как отказаться от изменений в файле (файлах) находящийся в рабочем каталоге (work directory)?
2.Как вернуть файл из индекса в рабочий каталог?
3.Как отменить ошибочный commit ?
4.Как создать новую ветку и переключится в нее?
5.Как создать git репозиторий из локального каталога на своем компьютере. Как его отправить на github.com?

В ответах на вопросы необходимо указать конкретные команда git.



1.  Для того чтиобы отказатся от изменений в рабочем каталоге есть команда
    git checkout .  - эта команда отменяет все изменения о которых git еще не знал.
    Если в конце команды точка то отменяются изменения во всех файлах с которыми 
    работали.
    А если нужно отменить изменения в определенном файле то вместо почки прописываем 
    имя того файла изменения в котором нужно отменить.

    Если надо удалить целый файл то для этого существует команда
    git clean  -xdf. Но сначала нужно посмотреть какие файлы будут удалены комондой
    git clean  -xdn и потом уже применнить команду git clean  -xdf

2. После команды git add. нащи файлы уже в индексе, чтобы это посмотреть примерим команду
   git status и увидим те файлы которые зеленым цветом, их можна вернуть обратно командой
   git restore --staget <name file>. Здесь тоже самое если пишем имя файла то возращает файл,
   а если точка в конще команды то возращает все файлы которые зеленым цветом.

3. Когда нужно удалить коммит то есть какая криминальная команда
   git reset HEAD~1 или  git reset HEAD^  Но лучше использовать 
   git revert HEAD тогда создастся зеркальный каммит и история будет сохранена

  А еще команда git commit --amend -m "здесь пишем коментарий каторый нодо изменить" - нам 
  изменит коментаний. И команда git commit  --amand -m "коментарий" + name file  - добавит
  файл.

4. Чтобы создать новуб ветку применяем команду
   git branch name - вместо name пишем название ветки
   git branch -v   - позволяет посмотреть какие ветки у нас есть
   git checkout -b test    - создает ветку test и переключает  на эту ветку
   git checkout main       - возвращает на ветку main
   git checkout test       - переключает на ветку test

5. Создаем папку, открываем ее с контекстного меню через Git Bash. Вводим команду 
   git init должна появится полупрозрачная папочка  git с точкой спереди.
   Cоздаем или копируем в эту папку файлы которые нужно отправить на гитхаб и потом команда
   git add .  - c точкой все файлы а если нужно какой то определенный файл то вместо точки 
   пишем имя этого файла. Kомандой git status смотрим в каком статусе эти файлы. 
   git commit -m "здесь пишем наш коментари - обезательно" - эта команда отправляет наши 
   файлы комитит и после команда
   git push и файлы на GitHab-e
